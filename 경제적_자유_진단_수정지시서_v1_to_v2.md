## 수정 1. P2-09 문항 재설계 (정답이 뻔한 문제 해결)

기존 P2-09를 찾아서 아래로 교체:

```
id: "P2-09"
prompt: "친구 두 명이 각각 사업을 시작한 지 1년이 됐습니다.\n첫째는 \"매달 매출이 꽤 나와\"라고 하고,\n둘째는 \"아직 매출은 작지만, 매달 남는 돈이 생겨\"라고 합니다.\n\n1년 뒤, 누가 더 유리한 위치에 있을까요?"
choices:
  A) "매출이 큰 첫째 — 규모가 크면 기회가 많다" → score: 2
  B) "남는 돈이 있는 둘째 — 실제로 돈이 쌓이고 있으니까" → score: 3
  C) "더 많은 정보가 필요하다 — 매출 크기나 비용 구조를 모르면 판단할 수 없다" → score: 3
  D) "사업은 해봐야 안다 — 1년으로는 판단하기 어렵다" → score: 1
```

선택지 4개 (기존 4개에서 유지). B와 C 모두 3점.

## 수정 8. 잠금 해제 방식 — 비밀번호(코드 입력) 방식으로 변경

### 8-1. 비멤버 2페이지 CTA 변경

기존 CTA 버튼("무료로 확인하기") 클릭 시 동작 변경:

- 기존: 카페 페이지 새 탭 열기 + LocalStorage에 unlocked 저장
- 변경: 카페 페이지 새 탭 열기 + **현재 탭에 코드 입력 화면 표시**

### 8-2. 코드 입력 화면 추가

CTA 클릭 후 현재 탭에 표시되는 화면:

```
[네이비 배경]

🔓 잠금 해제 코드를 입력하세요

카페에 가입하시면 환영 메시지에서
잠금 해제 코드를 확인할 수 있습니다.

[코드 입력 필드] (4자리 또는 6자리)

[잠금 해제하기] ← 골드 버튼

아직 가입 전이신가요?
[카페 가입하기 →] ← 텍스트 링크 (https://cafe.naver.com/sd161921)
```

### 8-3. 코드 검증 로직

```javascript
const UNLOCK_CODE = "BRIDGE";  // MH님이 나중에 변경 가능

function verifyCode(input) {
  return input.trim().toUpperCase() === UNLOCK_CODE;
}
```

- 코드 일치 시: LocalStorage에 unlocked 저장 + 잠금 해제 콘텐츠 표시
- 코드 불일치 시: "코드가 일치하지 않습니다. 카페 환영 메시지를 확인해주세요." 메시지

### 8-4. 이미 잠금 해제된 경우

LocalStorage에 unlocked가 true이면 코드 입력 없이 바로 잠금 해제 콘텐츠 표시.

### 8-5. 결과 공유는 코드 없이 가능

비멤버 1페이지(유형 + 레이더 + 간극)는 항상 공개. 공유 버튼도 코드 없이 작동.

## 수정 9. GA 추가

head 태그 안에 아래 GA4 gtag 스니펫을 추가:

```html
<script async src="https://www.googletagmanager.com/gtag/js?id=G-KJNV6DHSGC"></script>
<script>
  window.dataLayer = window.dataLayer || [];
  function gtag(){dataLayer.push(arguments);}
  gtag('js', new Date());
  gtag('config', 'G-KJNV6DHSGC');
</script>
```

기존 trackEvent 함수가 gtag를 호출하도록 연결되어 있는지 확인하고, 안 되어 있으면 연결:

```javascript
function trackEvent(eventName, params = {}) {
  if (typeof gtag !== 'undefined') {
    gtag('event', eventName, params);
  }
}
```
