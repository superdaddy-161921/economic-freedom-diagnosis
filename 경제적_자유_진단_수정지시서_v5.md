## 수정 1: 인사이트 브레이크 자동 넘김 → 탭 필수 [중간]

**문제:** 인사이트 브레이크(축 완료 후 설명 화면)가 자동으로 넘어가서 다 읽기 전에 사라짐.

**수정:** 인사이트 브레이크 화면에 자동 전환 타이머가 있다면 제거. 사용자가 "다음" 또는 "계속하기" 버튼을 탭해야만 다음 축으로 넘어가도록 변경. 버튼 스타일은 기존 골드 버튼과 동일.

인사이트 브레이크 화면 하단에 명확한 CTA 버튼 배치:
```html
<button class="btn-primary" onclick="continueAfterBreak()">계속하기 →</button>
```

`showInsightBreak` 함수 내에서 `setTimeout`이나 자동 `advanceQuestion` 호출이 있다면 제거하고, 위 버튼 클릭 시에만 다음 문항으로 진행되도록.

## 수정 2: 새 문항 렌더링 시 선택 상태 초기화 [쉬움]

**문제:** 다음 문항으로 넘어갔을 때 이전 문항에서 선택한 위치의 보기가 이미 선택된 상태로 표시됨.

**원인:** 문항 렌더링 시 `existing` 변수(928번 줄 부근)가 해당 문항에 대한 이전 답변이 있을 때 선택 상태를 복원하는데, 이것이 "앞으로 진행할 때"에도 작동하고 있음.

**수정:** 선택 상태 복원은 "이전 버튼으로 돌아갔을 때"에만 작동해야 함. 새 문항(아직 답변하지 않은 문항)으로 진행할 때는 모든 선택지가 미선택 상태여야 함.

```javascript
// 현재: 항상 이전 답변 복원
const existing = state.answers[q.id];
if (existing && existing.label === c.label) btn.classList.add('selected');

// 수정: 해당 문항에 답변이 저장되어 있을 때만 복원 (이전 버튼으로 돌아온 경우)
// 이 로직 자체는 맞지만, 문제가 다른 곳에 있을 수 있음:
// - selectChoice에서 답변 저장 시 label이 아닌 다른 키로 매칭되고 있는지 확인
// - 또는 CSS .selected 클래스가 DOM에서 제대로 초기화되지 않는지 확인

// 안전한 수정: renderQuestion 함수 시작 부분에서 모든 선택지 초기화
container.innerHTML = '';  // 이미 있다면 OK, 없다면 추가
```

**디버깅 포인트:** `renderQuestion` 함수에서 `container.innerHTML = '';`이 호출되는지 확인. 이것이 있으면 DOM은 초기화되는데, CSS 애니메이션이나 transition이 이전 상태를 유지하는 것일 수 있음. 브라우저 캐시 문제일 수도 있으니, 선택지 버튼 생성 시 `btn.classList.remove('selected')`를 명시적으로 추가.

## 수정 3: 하단 네비게이션 화면 고정 [쉬움]

**문제:** 일부 기기에서 "← 이전" 버튼과 "N / 34" 질문 번호가 화면 밖으로 밀려서 스크롤해야만 보임.

**수정:** 하단 네비게이션 바를 화면 하단에 고정.

```css
/* 하단 네비게이션 바 (이전 버튼 + 질문 번호가 있는 영역) */
.question-nav {
  position: fixed;
  bottom: 0;
  left: 0;
  right: 0;
  background: var(--navy);
  padding: 12px 20px;
  display: flex;
  justify-content: space-between;
  align-items: center;
  border-top: 1px solid rgba(255,255,255,0.1);
  z-index: 100;
}
```

본문 영역에 하단 패딩 추가 (고정 바에 가리지 않도록):
```css
.question-content {
  padding-bottom: 70px; /* 하단 고정 바 높이만큼 */
}
```

**주의:** 프로그레스 바는 상단에 있으므로 영향 없음. 하단 바에 포함되는 요소: "← 이전" 버튼 + "N / 34" 텍스트.

## 수정 4: P3-05 C선택지 명확화 [쉬움]

**문제:** "책, 강의, 스터디 등 체계적으로 배운 적 있다"에서 책만 읽은 사람이 "강의와 스터디는 안 했는데 해당되나?" 의문.

**현재 (P3-05, id:"P3-05"):**
```javascript
{label:"C", text:"책, 강의, 스터디 등 체계적으로 배운 적 있다", tag:"체계적"}
```

**변경:**
```javascript
{label:"C", text:"책을 읽거나, 강의를 듣거나, 스터디에 참여한 적 있다", tag:"체계적"}
```

## 수정 5: P2-05 복리 문항 전면 재설계 [높음]

**문제:** 숫자가 구체적이라 계산기로 풀게 됨. 복리 직관이 아니라 수학 테스트가 됨. 문항 구조 자체를 이해하지 못하는 사용자도 있음.

**현재 (P2-05, id:"P2-05"):**
```javascript
{ id:"P2-05", axis:2, prompt:'매달 50만원을 연 7% 수익률로 20년간 투자하면,\n넣은 돈의 총액은 1억 2,000만원입니다.\n\n20년 뒤 총액은 대략 얼마일까요?',
  choices:[
    {label:"A", text:"약 1억 5,000만원 (조금 더 불어남)", score:1},
    {label:"B", text:"약 2억 원 (거의 두 배)", score:2},
    {label:"C", text:"약 2억 6,000만원 (두 배 이상)", score:3},
    {label:"D", text:"감이 잘 안 온다", score:0}
  ]}
```

**변경:**
```javascript
{ id:"P2-05", axis:2, prompt:'두 사람이 같은 금액을 매달 투자합니다.\n첫째는 25살에 시작해서 35살에 멈췄고(10년간),\n둘째는 35살에 시작해서 65살까지 계속했습니다(30년간).\n\n65살이 됐을 때, 누가 더 많을까요?',
  choices:[
    {label:"A", text:"당연히 둘째 — 30년이 10년의 세 배니까", score:1},
    {label:"B", text:"둘째가 많겠지만 차이는 생각보다 작을 것이다", score:2},
    {label:"C", text:"비슷하거나 오히려 첫째가 많을 수도 있다", score:3},
    {label:"D", text:"잘 모르겠다", score:0}
  ]}
```

**점수 산출 영향:** 기존과 동일하게 0~3점 범위. 축2 최대 27점 구조에 변동 없음.
